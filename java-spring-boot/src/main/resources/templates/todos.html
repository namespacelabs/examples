Welcome to Namespace.

In this example, Namespace deployed a Java Spring Boot application that uses Postgres as a database.

The database is consumed as a Namespace resource provided by a shared Postgres provider. The resource produces a typed instance object which provides a password along with the endpoint for the Java application to consume. Another advantage is that resources have their own lifetime modeling and initialization only happens once. In the case of a database, this means that the schema will be applied as part of the lifecycle and our application does not need to worry about it.
